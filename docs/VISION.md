# 智能投标辅助系统 - 终极愿景与演进蓝图 (Project Vision & Endgame)

本文档旨在明确本项目的终极目标（Endgame），作为团队在重构与迭代过程中的“北极星”指南，确保所有架构设计和 Agent 演进都服务于这一终极蓝图。

## 🎯 核心一句话愿景

> **打造一个可商用、多租户的“AI 招投标自动驾驶系统（包含人工干预方向盘）”——从扔进一份几百页的 PDF 招标文件开始，到最终吐出一份可以直接拿去装订的、高质量的投标书 Word 终稿。**

---

## 🏔️ 必须跨越的四大终极形态 (The 4 Pillars of Endgame)

### 形态一：彻底的“端到端”自动生成闭环 (End-to-End Generation)
*   **当前痛点**：系统仅停留在“信息提取和仪表盘展示”层面，标书（技术方案、偏离表等）仍需人工手动编写。
*   **终极形态**：全面激活并接入 **Writer Agent（打字员/写手智能体）**。
    *   **动态排版**：LLM 能够根据招标文件中提取的“格式要求”，动态生成符合甲方规范的大纲目录。
    *   **偏离表生成**：汇总前序 Agent 的分析结果，自动生成毫无遗漏的《商务响应偏离表》和《技术响应偏离表》。
    *   **专业方案起草**：结合公司 RAG 知识库中的历史相似项目，自动起草针对当前痛点（如“彩钢瓦加固”）的技术解决方案与售后承诺书。
    *   **结果闭环**：前端提供一键导出 `.docx` 功能，AI 真正接管复制粘贴的繁杂劳动。

### 形态二：拥有“包工头”的真 Agent 架构 (Supervisor-based True Agent)
*   **当前痛点**：后端 LangGraph 是硬编码的、串行的流水线（Pipeline）。无并发、无条件跳转、无失败重试机制。
*   **终极形态**：引入基于 Structured Output 的 **Supervisor Agent（总控路由智能体）**。
    *   **动态分发 (Fan-out)**：总控拥有大局观，阅读完基础资料后，能自主决定并行唤醒哪些下游专业 Agent（如：侧重技术的标书多唤醒 Tech Worker，侧重价格的重点盯防 Cost Agent）。
    *   **容错与反思 (Worker Retry)**：如果某个子 Agent（如资质提取）反馈的结果残缺，Supervisor 能识别出“提取失败”，并携带新的 Prompt 引导其重试（最多重试2次），实现从“死板脚本”到“真智能体”的跨越。

### 形态三：“人机共驾”的审批流 (Human-in-the-loop)
*   **当前痛点**：AI 提取出致命风险（废标项、严重财务陷阱）后，仅在前端作为卡片展示，流程不受任何阻碍继续向下流转。
*   **终极形态**：在 LangGraph 的核心流转节点中设置强拦截（Interrupt）。
    *   **红线暂停**：一旦识别到致命废标项，或成本核算超出公司利润底线，AI 处理流**强制挂起**。
    *   **审批模态框**：前端全局弹窗报警，强制要求项目经理级别（人工）介入，阅读溯源原文并点击“确认风险并继续”或“驳回”，流程方可恢复（Resume）。这是商用级风控系统的底线。

### 形态四：万物皆可溯源的企业级 SaaS (Traceable Multi-Tenant Platform)
*   **当前痛点**：大模型固有的“幻觉”会导致用户对 AI 提取出的风险和价格产生不信任。数据缺乏严格的权限边界。
*   **终极形态**：建立不可撼动的信任感。
    *   **数据绝对隔离**：严格的多租户（Tenant）权限体系。A公司的用户永远只能调用A公司的内部指导价库、业绩库和资质库。
    *   **精准的高亮溯源**：AI 输出的每一个结论（无论是风险提示、还是资质缺漏），都必须附带 `exact_quote` 和 `page_num`。用户点击卡片，右侧的 PDF 预览器立即滚动高亮至原文件的特定段落，实现 100% 的证据链闭环。
