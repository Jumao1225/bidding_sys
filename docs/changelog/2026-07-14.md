## 追加改动（下午）：修复前端"提取文本"展示重叠内容

### 问题根因
`tasks.py` 中 `extracted_text` 字段由数据库 `DocChunk` 逐条拼接生成，  
而每个 Chunk 的 `content` 本身包含 `CHUNK_OVERLAP`（重叠区约 200 字符），  
导致前端"提取文本"视图出现重复段落。

### 修复方案：建立 `output.md` 路径三段传递链路

1. **`extractor_service.py`**（`parse_with_mineru`）  
   将 `mineru_service.parse_file()` 返回的 `md_file_path` 注入每个 `Document.metadata["md_file_path"]`。

2. **`parser_worker.py`**  
   从第一个 chunk 的 `metadata` 取出 `md_file_path`，  
   写入 `document.parsed_metadata["md_file_path"]` 随文档记录一起持久化到数据库。

3. **`tasks.py`**  
   优先读取 `document.parsed_metadata["md_file_path"]` 指向的 `output.md` 原始文件，  
   作为 `extracted_text` 返回给前端，彻底消除重叠文本。  
   仅在文件不存在时降级回 Chunk 拼接（并打印 warning 日志）。

### 改动文件
- `app/services/extractor_service.py`: `parse_with_mineru` 注入 `md_file_path` 到 Document metadata。
- `app/agents/nodes/parser_worker.py`: 提取并持久化 `md_file_path` 到 `parsed_metadata`。
- `app/worker/tasks.py`: 优先从 `output.md` 读取原始 Markdown 全文供前端展示；新增 `db.expire_all()` 修复 SQLAlchemy Identity Map 缓存问题。

## 追加改动（下午二）：Word 文档表格标题识别

### 问题背景
Word 文档中表格标题通常是表格上方的独立段落（如"表1 资质要求评分"），
`python-docx` 顺序迭代时将其作为普通段落输出，Markdown 中缺乏与表格的视觉关联。

### 修复方案：前瞻检测（Look-ahead）+ 多规则判定
在 `mineru_service._convert_docx_to_markdown` 中实现：

**`_is_table_caption(text, paragraph)` 静态方法**（满足任一规则即判定为表格标题）：
1. 文本以「表」+ 数字/汉字序号开头（`^表\s*[\d一二三四...]+`）
2. 段落 style 名称包含 `caption` / `表题` / `table caption`
3. 文本 ≤60 字且段落所有有效 Run 均为粗体

**`_convert_docx_to_markdown` Look-ahead 逻辑**：
- 将 `body.children` 收集为列表，按索引迭代
- 每个段落处理时，向后扫描跳过空行，遇到表格则调用 `_is_table_caption` 判断
- 判定为表格标题的段落以 `\n**标题文本**` 格式输出（加粗），建立明确的视觉关联

### 改动文件
- `app/services/mineru_service.py`: 新增 `_is_table_caption` 静态方法；重构 `_convert_docx_to_markdown` 实现前瞻检测。
