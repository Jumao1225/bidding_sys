# 2026-07-21 改动记录

## 1. 核心功能与业务逻辑
- **前端履约盘点数据真实化 (UploadBox.tsx)**
  - 移除了上传解析时硬编码的模拟公司资质（`mockCompanyQuals`）。
  - 新增了在触发分析前，动态调用后端 `GET /api/v1/qualifications/` 接口拉取真实资质数据。
  - 将拉取到的资质列表拼接并格式化为完整的公司资质背景描述字符串（包含公司名称、资质等级、到期时间等），作为 `company_quals` 传递给后端。
  - 确保了 AI 大模型在进行履约盘点（匹配度打分与分析）时，完全基于用户在“资质中心”内实际管理的真实数据进行逻辑判断。

## 2. 系统配置与安全
- **后端 CORS 跨域配置优化 (config.py)**
  - 在 `BACKEND_CORS_ORIGINS` 许可白名单中新增了 `http://127.0.0.1:5173` 和 `http://127.0.0.1:5174`。
  - 避免了部分本地前端开发环境（如 Vite 默认使用的 127 域名）在调用 API 接口时遭遇浏览器的 CORS 拦截报错。
