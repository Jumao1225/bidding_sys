## 10. 系统底层架构稳定性与兼容性修复
- **彻底拔除 Celery，降级使用原生 BackgroundTasks (worker/tasks.py, endpoints/analysis.py)**
  - 移除了 Celery 的架构依赖（解决 Windows 平台上 Celery Worker 子进程初始化 HuggingFace Transformers Tokenizer 时的严重多进程死锁/静默崩溃顽疾）。
  - 将 `analyze_bidding_doc` 长耗时解析任务无缝切换至 FastAPI 原生的 `BackgroundTasks`，在 Uvicorn 的主进程后台纯 Python 线程池中安全执行。
- **解决 HuggingFaceEmbeddings 内存溢出 (OOM) 导致进程暴毙 (services/llm_service.py)**
  - 在大文本块批量嵌入（Embedding）阶段，精准定位了 PyTorch 抛出的 C++ 底层 `DefaultCPUAllocator` 内存超限错误（1.2GB瞬时峰值）。
  - 将 `bge-m3` 推理时的 `batch_size` 优化参数从 `32` 激进下调为 `4`，以时间换空间，成功压平了内存消耗峰值，保障了后台线程的极高稳定性。
  - 同时追加了 `KMP_DUPLICATE_LIB_OK="TRUE"` 与 `TOKENIZERS_PARALLELISM="false"` 环境变量，进一步封堵了因 OpenMP 和 Rust 底层线程复用在 Windows 下引发的冲突雷区。
