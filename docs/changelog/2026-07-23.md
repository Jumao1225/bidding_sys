# 项目改动日志 - 2026-07-23

## 📋 变更概述
本日主要完成了**局域网多主机访问支持**、**前端 API 动态地址与防错解析机制**、**Vite 反向代理优化**以及**Windows 防火墙端口放行排查**，解决了其他主机在局域网下无法正常访问系统及登录接口报 404 / 超时（TIMED_OUT）的问题。

---

## 🔧 详细改动清单

### 1. 局域网访问与 Vite 开发服务器配置 (LAN Access & Dev Server)
* **`frontend/package.json`**
  - 更新启动脚本 `"dev"` 为 `vite --host 0.0.0.0`，允许 Vite 监听服务器本机的所有网卡接口。
* **`frontend/vite.config.ts`**
  - 增加 `server.host = '0.0.0.0'` 配置。
  - 配置 `server.proxy` 反向代理规则：将 `/api` 与 `/uploads` 路径的请求自动转发至本地后端 `http://127.0.0.1:8000`，提供本地代理兜底保障。

### 2. 前端 API 动态解析与容错补全 (API Base URL & Fault Tolerance)
* **`frontend/src/utils/api.ts`**
  - 新增 `getApiBaseUrl()` 辅助导出函数。
  - 增加防错规范化逻辑：若 `.env` 中 `VITE_API_BASE_URL` 未写 `http://` 前缀或端口（如写为 `192.168.10.190`），函数将自动补充为 `http://192.168.10.190:8000`；若为空则自动退回相对路径（走 Vite 代理）。
* **前端全量页面与组件解耦**
  - 替换了项目中 10 个关键页面/组件里硬编码的 `'http://127.0.0.1:8000'` 地址，统一采用 `getApiBaseUrl()` / 相对路径。
  - 覆盖文件包括：
    - `src/pages/Login.tsx`
    - `src/pages/Home.tsx`
    - `src/pages/AnalysisDashboard.tsx`
    - `src/pages/QualificationCenter.tsx`
    - `src/pages/PriceBookCenter.tsx`
    - `src/pages/admin/SystemAdmin.tsx`
    - `src/components/UploadBox.tsx`
    - `src/components/qualifications/QualificationUploadModal.tsx`
    - `src/components/DraftCard.tsx`
    - `src/components/ChatPanel.tsx`

### 3. 环境配置与跨域放行 (Environment & CORS)
* **`.env`**
  - 设置 `BACKEND_CORS_ORIGINS=["*"]`，放行局域网中不同客户端 IP 发起的跨域请求。
  - 规范并标注 `VITE_API_BASE_URL` 变量的使用方法。

### 4. 系统网络与防火墙配置指南 (Network & Firewall Guide)
* 梳理并排查了 Windows 防火墙拦截外部访问 TCP 8000 端口导致客户端 `ERR_CONNECTION_TIMED_OUT` 连接超时的网络层原因。
* 明确了通过 PowerShell `New-NetFirewallRule` 永久放行入站 8000 端口的操作命令与标准。

### 5. Copilot 聊天 Agent 文档上下文同步修复 (Chat Agent Context Synchronization)
* **`frontend/src/pages/AnalysisDashboard.tsx`**
  - 在仪表盘挂载与路由参数 `:id` 变更时，增加自动将当前有效文档 ID 写入 `localStorage.setItem('bidding_document_id', ...)` 并触发全局 `bidding_document_changed` 事件的机制。
  - 在历史数据或者 API 返回最终 `document_id` 时再次校验并实时同步广播。
* **`frontend/src/components/ChatPanel.tsx`**
  - 引入动态 `activeDocId` 状态，兼顾 `prop` 传入与监听 `bidding_document_changed` 全局事件，彻底解决由于页面切换或直接打开历史解析 URL 时 Copilot 提示“未上传或选择标书”且拦截对答的问题。
* **`backend/tests/api/test_chat.py`**
  - 新增 Chat API 接口单元测试，覆盖参数拦截校验、不存在文档 403 权限拦截以及标准 SSE 响应流模式，测试全部通过（3 passed）。

## 🎯 验证结果
- **局域网连通性**：同局域网下其他主机（如 `192.168.10.147`）顺利访问服务器（`192.168.10.190:5173`）前端页面。
- **登录与数据交互**：跨主机成功完成登录鉴权并正常获取解析历史数据，无 CORS 拦截与 404/Timeout 错误。
- **Copilot 对答功能**：打开任意已解析或历史文档，Copilot 助手均能即时识别当前处理的文档，顺利响应多轮对话与 RAG 检索提问。
