## 12. 修复前端术语歧义、财务精度与 Agent 盘点不稳定性 (Fullstack)
- **文件**: `backend/app/agents/nodes/strategy_agent.py`, `frontend/src/components/dashboard/FinancialCard.tsx`, `frontend/src/components/dashboard/TimelineCard.tsx`
- **改动**:
  1. **盘点维度稳定性 (Strategy Agent)**: 优化了 `StrategyAgent` 的 `Prompt`，下达最高指令要求大模型**严格一对一**对照提取出的核心硬性门槛进行分析，禁止随意合并或发散。彻底解决了之前由于 Prompt 自由度过高导致“履约盘点”每次刷新时，要求条目数量和内容频繁变动的不稳定现象。
  2. **核心财务精度 (FinancialCard)**: 移除了前端为了排版美观而对超万金额进行的 `/ 10000` 并保留两位的四舍五入逻辑。统一采用千分位展示并精确到“分”（`元` 为单位），彻底消除了因四舍五入造成的招投标金额致命误差隐患（如 `118.145万` 被错误截断）。
  3. **专业术语兼容 (TimelineCard)**: 将前端界面的 `工期` 标签更改为兼容性更强的 `交付/工期`，消除了用户在处理纯设备/货物采购标（交货期）时产生的业务术语困惑。
