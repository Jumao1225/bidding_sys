# 2026-07-09 重构与架构升级更新记录 (Changelog)

## 1. 核心架构重构 (Backend Architecture)
- **全面实施 DDD + 多智能体架构**：将原本揉杂的业务逻辑严格拆分到了 `api/`, `core/`, `db/`, `agents/`, `skills/`, `graph/`, `worker/`, `schemas/` 分层中。
- **LangGraph 工作流编排**：
  - 废弃松散的函数调用，使用 LangGraph 定义 `BiddingState` 全局状态图。
  - 将各个大模型步骤（资质分析、风险排查、成本核算）统一封装为符合 `_agent.py` 命名规范的智能体节点（如 `strategy_agent.py`, `cost_agent.py` 等）。
- **技能组件轻量化**：废弃了繁重的 `BaseSkill` 面向对象模式，统一改用轻巧的 LangChain `@tool` 装饰器。
- **高可用与重试机制**：引入 `tenacity`，为大模型调用 (`generate_structured_json`) 加入了带有指数退避算法的自动重试机制兜底。

## 2. 异步改造与流式推送 (Asynchronous & SSE)
- **引入 Celery 后台队列**：彻底解决了大文件 PDF 导致 HTTP 请求超时的问题。现在 `/api/v1/analysis/upload-and-analyze` 仅负责快速派发任务并返回 `task_id`。
- **SSE 长连接进度订阅**：新增 `/api/v1/sse/progress/{task_id}` 接口。后台 Agent 节点在执行时会通过 Redis 发布/订阅模型，将状态进度实时推送给前端。

## 3. 测试隔离层标准化 (Testing Refactoring)
- **规范化测试层级**：彻底重组了 `backend/tests/`，建立了标准的 `unit/` (单元测试)、`integration/` (集成测试) 和 `api/` (接口测试) 结构。
- **Fixture 数据解耦**：将所有硬编码的大模型 Mock JSON 数据抽离至专门的 `fixtures/mock_responses.json` 中，在 `conftest.py` 或测试用例内动态加载。
- **异步接口测试支持**：安装配置 `pytest-asyncio` 与 `httpx.AsyncClient`，完全替换了过时的 `Starlette TestClient`，5 条核心测试用例 100% 通过。

## 4. 前端 Epic Design 旗舰视觉升级 (Frontend UI/UX)
- **原生 SSE 通信打通**：重构 `UploadBox.tsx`，前端通过 `EventSource` 接管后台进度，实现了实时加载、动态阶段展示，消除了“黑盒等待”焦虑。
- **顶级 SaaS 设计美学 (Aesthetics)**：
  - **环境构建**：`index.css` 引入 `Inter` 字体，并构建了带有 `radial-gradient` 极光流动的空间网格背景。
  - **组件质感重塑**：全局大量引入 **毛玻璃 (Glassmorphism)** 效果（`backdrop-blur`、暗黑与冰霜变体），搭配 `hover:-translate-y-1`、内部渐变进度条等丰富微交互。
  - **Cinematic Banner**：将无聊的欢迎区域改造成具有 2.5D 几何点阵叠加与悬浮光晕的次世代引导横幅。
  - **拟真聊天气泡**：为 `ChatPanel.tsx` 增加了类人打字机点阵动画 (Typing Indicator) 与对话背景光晕。

## 5. 基础设施与持久化升级 (Database & Infrastructure)
- **本地 Docker 混合部署**：重写了 `docker-compose.yml`，一键拉起纯洁的 PostgreSQL 和 Redis 服务，让业务代码在本地跑的同时无缝对接真实组件库。
- **强制阻断 SQLite 退化**：在 `backend/app/core/config.py` 与 Alembic 配置文件中加入了安全防御机制。一旦检测不到合法的 `DATABASE_URL` 将直接抛出 `ValueError` 而不是悄悄退化为 SQLite。
- **Alembic 初始化打通**：通过 `dotenv` 提前注入环境变量，完美解决 Pydantic v2 `Settings` 初始化时的取值异常问题。目前系统已全量执行了 `upgrade head`，原定架构中设计的多租户表、DocChunk表 等结构均已在 Postgres 容器中落地。
