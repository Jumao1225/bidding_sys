# 2026-07-10 代码改动记录

## 1. 前端 UI 与交互大重构 (Frontend UI/UX Revamp)
**涉及文件：** `App.tsx`, `UploadBox.tsx`, `ChatPanel.tsx`, `LocalDocxRenderer.tsx`

### 1.1 智能助手悬浮窗 (Floating Chatbot)
- 移除了原有的左右分栏硬编码，将智能解析助手 (`ChatPanel`) 重构为**全局悬浮窗**。
- 引入了 `react-draggable` 库，实现了悬浮球的自由拖拽。
- 增加了**智能边界计算与防越界逻辑** (Clamping)：根据悬浮球在屏幕上的坐标，动态计算对话框的展开方向（上半屏向下展开，下半屏向上展开），彻底解决了对话框在屏幕边缘被截断的问题。

### 1.2 灵活的结果分屏布局 (Resizable Split Layout)
- 废弃了标书分析结果页面固定的 1:1 布局，引入了 **“6:4 黄金分割”** 默认布局。
- 新增了**可拖拽的中轴线**，用户可以通过鼠标拖拽自由调节“原文件对照区”与“风险大盘”之间的比例，适应不同宽度的标书排版。
- 在原文对照区新增了**“沉浸阅读模式”（全屏开关）**，点击后可让文档预览占据 100% 的宽度，方便阅读字号较小或包含复杂表格的标书。

### 1.3 解决 Word 文档预览截断与滚动冲突 (Docx Preview Fix)
- 修复了因为外部容器 `h-[600px]` 限制与内边距 `p-6` 导致的文档预览底部被“一刀切”截断且不出现滚动条的 Bug。
- 在 `LocalDocxRenderer.tsx` 中重新启用了 `inWrapper: true` 配置，利用底层库自身的完整阅读器模式（带灰色背景、居中页面与原生滚动条）渲染原文档，大幅提升阅读体验。

### 1.4 状态持久化与流程重置 (State Persistence & Reset)
- 为了防止用户误刷新页面导致分析结果清空，在 `UploadBox.tsx` 中加入了 `localStorage` 缓存机制。现已支持缓存 `result`、`taskId` 以及用户的视图布局偏好。
- 在页面头部新增了 **“重新解析新文档”** 按钮，点击可一键清空本地缓存与当前状态，重新进入上传模式。

## 2. 后端稳定性修复 (Backend Robustness)
**涉及文件：** `backend/app/agents/nodes/strategy_agent.py`

### 2.1 修复 LLM 返回格式导致的解析崩溃
- **问题描述**：大模型在执行风险识别时，偶尔会不按 Prompt 约定返回字典 `{"risks": [...]}`，而是直接返回一个纯列表对象 `[...]`，导致后续调用 `.get()` 方法时抛出 `AttributeError` 并引发 500 错误。
- **解决方案**：在 `identify_risks_node` 中添加了健壮的类型断言（Type Checking）。如果检测到 LLM 返回的是 `list`，则直接赋值；如果是 `dict`，则安全地获取 `"risks"` 字段；其他异常情况则默认返回空列表，确保多智能体工作流在不稳定的 LLM 输出下依然不中断。
