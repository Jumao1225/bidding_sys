# Bidding Sys 日常改动日志 - 2026-07-16

## Agent 思考终端 (AgentTerminal) UI/UX 深度优化

### 问题背景
原有的 `AgentTerminal` 虽然实现了基础的后端 SSE 日志流接收与展示，但视觉呈现较为简陋，仅使用基础的文本和 Emoji，无法体现系统“智能”和“科技”的属性。且针对 Agent 复杂的回调参数（如 JSON 和工具函数名）缺乏语法高亮，可读性不佳。

### 优化方案
1. **引入高级图标系统**：新引入 `lucide-react` 组件库，将简陋的 Emoji 替换为高质量的终端、设置、成功、失败等科技感图标。
2. **语法智能高亮 (Syntax Highlighting)**：开发了 `HighlightedText` 解析组件，使用正则表达式动态捕获日志字符串中的 JSON 结构和特定的工具函数关键字（如 `extract_qualification_info`），并为其附加带有阴影和科技边框的代码块样式，提升日志专业度。
3. **科技感扫描动画 (Scan Effect)**：在 `index.css` 中注入全新的 `@keyframes scan` 动画。当解析处于 `isAnalyzing=true` 状态时，终端背景会自动渲染出雷达上下扫光的极客效果。
4. **悬浮交互细节 (Micro-interactions)**：为每条日志行添加 Hover 态反馈，显示行首指示箭头、增加行内发光投影，并在行尾渐现伪时间戳序列号，进一步强化专业后台终端的沉浸感。

### 改动文件
- `frontend/package.json`: 新增 `lucide-react` 依赖包。
- `frontend/src/index.css`: 新增 `scan` 动画 keyframes。
- `frontend/src/components/dashboard/AgentTerminal.tsx`: 全面重构组件，应用更深邃的科技色板和发光特效，并接入高亮组件。

3. **本地聊天记录持久化**：在 `ChatPanel` 内实现基于 `localStorage` 的按文档 ID 缓存，刷新不再丢失聊天上下文。

## 策略智能体数据流转重构与大模型降本增效

### 问题背景
1. **数据重复生成与幻觉风险**：此前的 Master Agent 在调用 5 个专业的垂直提取工具后，还会强行输出一个包含核心字段和 `pain_points` 的 JSON，增加了无谓的 Token 消耗且容易产生大模型幻觉。
2. **下游未充分利用结构化数据**：后端的策略智能体（Strategy Agent）在进行“履约盘点”和“风险提示”分析时，没有直接去读取 5 大专业提取工具存入的结构化 Metadata，而是依赖 `parsed_metadata` 里简单的字符串或进行高成本的 RAG 盲搜。
3. **前端历史记录缺失核心面板**：在修复了本地缓存遗留问题后，历史记录页面的 `UploadBox` 被意外隐藏，导致从首页点击历史解析记录进入时，无法查阅原始文档和履约盘点面板。
4. **同名文件解析互相覆盖**：此前系统在处理同名文件上传时，会直接删除旧文件记录，导致版本覆盖和重复解析。

### 优化方案
1. **Master Agent 纯粹化**：彻底移除主控智能体最后强制输出 JSON 总结的逻辑，让其蜕变为纯粹的“工具调度器”。删除了冗余的 `pain_points` 字段，并同步移除了遗留的测试用例与提取脚本。

3. **高阶分析结论深度落盘**：将大模型生成的 `qualifications_analysis` 和 `risks_analysis` 复杂 JSON 报告，在任务结束时序列化入 `Document.parsed_metadata`，为历史“秒开”打下基础。

5. **智能 MD5 缓存去重**：更新文件保存逻辑，使用 MD5 Hash 取代单纯的文件名比对。完全一致的文件直接复用旧解析进度，内容有修改的同名文件则单独存为新版本记录，确保解析版本隔离。
6. **完善项目架构文档**：新建了 `docs/data_flow_architecture.md`，专门记录这种复杂大模型工作流在“生产 -> 存储 -> 渲染”三大阶段的链路逻辑。

## Master Agent 提示词与工具链深度调优 (Prompt Engineering)

### 问题背景
1. **大模型上下文窗口浪费**：原 Prompt 中硬编码了 `【标书头部内容 (前25000字)】`，将海量未处理的原文直接喂给调度器，导致每次 API 调用浪费上万 Token。
2. **串行调用耗时过长**：原指令强制要求“逐一调用这 5 个专项工具”，导致大模型进行多次 `Thought -> Action -> Observation` 循环，整体响应极其缓慢。
3. **残留的无用兜底工具**：在系统升级结构化存储后，通用 RAG 检索工具 (`search_bidding_document`) 提取的文本由于没有对应的数据库字段接收，对于 Master Agent 来说成为了“查了也没法存”的无效工具。

### 优化方案
1. **彻底解除大模型文本负载**：从 Prompt 中删除了 25000 字的原文变量，Master Agent 仅依靠文档 ID 和章节大纲（TOC）作为路标进行思考。具体的原文检索任务全权下放给 5 个内部集成了 RAG 的专项工具，极大降低了 Token 消耗。
2. **升级并发函数调用 (Parallel Tool Calling)**：修改指令为“尽可能**并发（一次性）**调用这 5 个专项工具”。大模型现在可以瞬间发射 5 个工具请求，并仅对返回 null 的“哑弹”工具进行定向单点重试，解析速度迎来质的飞跃。
3. **剔除无用冗余工具与遗留指令**：从 `supervisor.py` 的可用工具列表 `all_tools` 中彻底拔除 `search_bidding_document`，断绝其发散可能，确保完全按照方案 A（后台静默流水线仅负责结构化基建，自由探索交由前台 Chat Agent）运行。同时清除了 Prompt 中残留的关于“挖掘痛点工况”的干扰指令。
4. **运行时大纲日志 (TOC Logging)**：增加了后台日志记录，在运行时显式打印 Document Parser 生成的完整章节大纲，便于开发者监控大模型的“视野”。

### 改动文件
- `backend/app/agents/supervisor.py`: 深度重构 Prompt，实施并发调用指令，剔除冗余文本变量，并从 `all_tools` 中卸载通用检索工具。

3. **极简化路由 (Router 层)**：将 `chat.py` 彻底清理至仅剩不到 50 行，专门负责 HTTP Request 的入参校验及向 `chat_agent` 的流式转发，不再染指任何具体业务。

## ChatPanel 前端 Markdown 渲染与排版修补 (UI/UX Hotfix)

### 问题背景
在接入 Chat Agent 后，由于其输出了包含表格 (`|...|...|`) 的 GitHub Flavored Markdown 语法，而原有的 `react-markdown` 基础组件不仅不支持该语法，且前端项目中缺少对应的 Tailwind 排版样式集，导致表格呈现为毫无样式的原始纯文本，极大影响了阅读体验。

### 优化方案
1. **渲染器能力补全**：安装并引入了 `remark-gfm` 插件，将其挂载到 `ReactMarkdown` 的 `remarkPlugins` 中，赋予其解析原生 Markdown 表格、删除线等扩展语法的能力。
2. **全局排版引擎激活**：通过 npm 安装 `@tailwindcss/typography` 官方插件，并在 `index.css` 的核心入口注入 `@plugin "@tailwindcss/typography";` 激活了 `prose` 样式族，使得前端的所有富文本和 Markdown 解析立马拥有了现代化的精美排版基准。
3. **表格专属美化**：在 `ChatPanel.tsx` 中为对应的消息气泡外层追加了 `prose-table:w-full prose-table:border-collapse prose-th:border-b-2` 等一系列原子化样式覆盖，让最终呈现的表格边框清晰、间距舒适、自带精美排版。

### 改动文件
- `frontend/package.json`: 新增 `remark-gfm` 和 `@tailwindcss/typography` 依赖包。

## 全局悬浮组件拖拽体验极速优化 (Drag & Drop UX/Performance Overhaul)

### 问题背景
之前系统中无论是悬浮的机器人按钮（FAB）还是展开的 ChatPanel，其拖拽手感都存在严重缺陷：
1. **拖放回弹 (Snapping Bug)**：FAB 按钮被设定了强制的边缘吸附逻辑，导致用户试图将其放置在屏幕中间时会被强行弹回边缘。
2. **原生拖放冲突 (Native Drag Interference)**：在包含 SVG 的元素上触发拖拽时，浏览器会高频截胡并启动原生的“图片拖放”操作（Native Drag & Drop），导致 `mousemove` 事件链断裂，出现“拖不动”或“点一次就死锁”的诡异现象。
3. **严重掉帧与主线程阻塞 (Layout Thrashing & Lag)**：拖动时每一像素的移动都会触发 React `setState` 更新全屏组件，同时在修改 `left` / `top` 时引发了浏览器海量的 Layout Recalculation（重排）。在叠加了 `backdrop-blur` 滤镜的 ChatPanel 背后，这种密集的重绘导致了肉眼可见的极度卡顿（不丝滑）。

### 优化方案
1. **废除边缘强制吸附**：移除了 FAB 在 `handleMouseUp` 时的边缘吸附代码，实现全屏幕自由停放。
2. **拦截原生拖拽操作**：使用最标准的 HTML5 方案，通过向容器注入 `onDragStart={(e) => e.preventDefault()}` 彻底屏蔽浏览器的多管闲事，同时剔除了之前错误使用的 `mousedown` 阶段的 `preventDefault()`（该错误导致了 `onClick` 无法触发）。
3. **电竞级 60FPS 重构 (RAF + GPU 硬件加速)**：
   - **完全绕过 React 渲染引擎**：拖拽期间不再调用 `setState`，而是利用原生的 `document.addEventListener` 闭包和 DOM 直接修改，仅在鼠标松开的瞬间同步最终状态给 React。
   - **节流同步刷新率**：核心移动逻辑被包裹在 `requestAnimationFrame` 中，保证无论鼠标反馈有多快，样式更新都严格对齐显示器的物理刷新率。
   - **强制 GPU 独立图层**：为所有可拖拽组件附加了 `will-change: transform` 属性，并将原本修改 `left/top` 的坐标重算，全部升级为了基于 GPU 的 `transform: translate(x, y)` 矩阵变换，彻底终结了布局回流（Reflow）引起的掉帧。

### 改动文件
- `frontend/src/App.tsx`: 移除全部陈旧的 `useRef` 全局监听缓存，采用动态绑定的原生理智闭包。使用 RAF 和 GPU Transform 全面重写 ChatPanel 与 FAB 的双向拖拽引擎。

## 全局悬浮组件交互体验打磨与原文件预览 Bug 修复 (UX Polish & Bugfixes)

### 问题背景
在重构了基于 GPU 硬件加速的 60FPS 拖拽引擎后，暴露出了一些边缘场景下的交互与展示 Bug：
1. **死板的坐标系截断**：小屏幕或移动悬浮球到屏幕中间时，面板展开依然强制依赖 `bottom/right` 锚点，导致面板大半部分飞出屏幕，产生不可视的截断现象。
2. **面板跟随脱离（幽灵漂移）**：拖拽过面板产生偏移后，再次拖动悬浮球时，因为历史偏移量未清空，面板会直接飞向未知的屏幕外空间。
3. **滚动失效（幽灵遮挡）**：在面板关闭状态下，其 DOM 的 `pointer-events-auto` 属性依然反向击穿了外层容器，形成了一个看不见的“鼠标黑洞”，导致页面局部无法滚动。
4. **历史记录原文件预览空白**：从仪表盘恢复历史标书记录时，由于缺少原文件扩展名，且后端下载接口未适配，导致预览插件一直处于白板状态。
5. **重叠与尺寸计算异常**：引入了重复的常量声明导致 Vite 崩溃；且原尺寸过大容易遮挡视线。

### 优化方案
1. **完美的边缘滑动与尺寸动态适应算法**：
   - 废弃死板的定位，重新设计坐标系渲染方案。使用 `Math.max(20, Math.min(...))` 精准计算视口可用空间，确保不管悬浮球在哪，面板一旦触及屏幕边缘，就会像涂了润滑油一样平滑贴边。
2. **重置拖拽历史与打断保护**：
   - 当释放悬浮球的瞬间，强制清空面板的拖拽偏移量，保证面板能随时完美重置到悬浮球附近的最佳安全位置。
3. **智能边缘吸附（Edge Snapping）回归**：
   - 像 iOS AssistiveTouch 一样，引入了释放后智能吸附到左右边缘的弹簧果冻动画，并支持“动画中途无缝打断”。
4. **彻底消灭幽灵遮挡**：
   - 将内层的 `pointer-events-auto` 设为严格依据 `isChatOpen` 的条件渲染，彻底释放页面滚动控制权。
5. **历史原文件预览完美复现**：
   - 修改后端下载接口：增加 DB 依赖，优先判断是否为历史文档 ID 以直连数据库返回物理路径。
   - 修改前端逻辑：后端追加返回 `filename`，前端 `viewerDocuments` 支持从中拆解后缀，重新激活 DocViewer 组件。
6. **精致瘦身与代码清理**：
   - 将悬浮球尺寸从 64px 缩小到精致的 56px，清理冗余常量。

### 改动文件
- `frontend/src/App.tsx`: 深度打磨拖拽计算公式，增加边缘吸附逻辑，修复 Pointer Events 遮挡与编译报错。
- `frontend/src/components/UploadBox.tsx`: 修复了 `DocViewer` 扩展名缺失导致白板的 Bug。

## 原文对照区分屏拖拽排版与性能极致优化 (Split-View UX & Performance Polish)

### 问题背景
1. **滚动条与选词失效（幽灵遮挡）**：在分屏原文对照区，为了增加拖拽分割线的可点击热区，为其设置了 `-mx-3`（左右负边距），这导致了一个 24px 的透明层直接覆盖在了左侧面板的滚动条上。用户试图拖动滚动条或在右边缘选中文字时，鼠标事件全部被拖拽中轴线拦截，导致交互冻结。
2. **极度卡顿与误选文字（重绘雪崩）**：拖拽分割线时，之前是通过 React 的 `setSplitRatio` 状态更新来驱动宽度变化。由于每次鼠标移动（哪怕 1 像素）都会触发庞大的 `UploadBox` 组件（包含了复杂的 `react-doc-viewer` 引擎）完整重绘，导致严重的掉帧。同时，由于拖拽时未禁用文本选中，极易误高亮大片文字。

### 优化方案
1. **纯物理宽度的安全隔离布局**：
   - 废除了取巧的负边距方案（移除 `-mx-3`），让拖拽线成为一个纯粹且独立的 24px 宽度的 flex 元素。
   - 通过 CSS `calc()` 动态计算左右面板的百分比宽度并各自向内退让 12px（例如：`calc(splitRatio% - 12px)`），确保 24px 的拖拽感应区被安全物理隔离，绝对不会再去覆盖左右两边任何一毫米的滚动条或文本。
2. **电竞级 60FPS 顺滑拖拽与防误触护盾**：
   - 彻底废弃在 `mousemove` 中调用 `setState`，改为通过 `useRef` 获取子面板的 DOM 节点。
   - 引入 `requestAnimationFrame` 硬件级节流调度，直接高频操作 DOM 样式（`style.width`），实现完全绕过 React 生命周期的高达 60 帧如丝般顺滑拖拽，仅在拖拽结束（`mouseup`）时才进行一次状态同步以兜底。
   - 在拖拽发起的瞬间，向 `body` 注入 `user-select: none` 以杜绝文字误选；并同时对左右面板开启临时的 `pointer-events: none` 护盾，防止里层的 PDF 或 DOM 节点意外吞噬鼠标事件。

### 改动文件
- `frontend/src/components/UploadBox.tsx`: 重新设计了中间分割线的布局逻辑和宽度 `calc()`，并使用 RAF 与原生 DOM 样式重构了整个拖拽响应引擎。

## 前端渲染与动画极致性能优化 (Frontend Rendering & Animation Overhaul)

### 问题背景
1. **拖拽重排卡顿**：之前的分屏拖拽由于直接修改 DOM 宽度，在处理极其庞大的长文档 DOM 树时，会引起严重的重排卡顿。
2. **长文本模式内存爆炸**：在纯文本模式下，大量的多颜色高亮标签在渲染几十万字长文本时会阻塞主线程，导致页面失帧。
3. **原文档引擎拖累全局**：PDF 或 Word 加载后生成的海量节点拖慢了整体操作，且缺少真正的分页按需加载。

### 优化方案
1. **物理引擎级动画升级**：引入 `framer-motion` 接管基础的 Tailwind 动画，赋予卡片与弹窗更加自然顺滑的物理回弹效果。
2. **专业面板组件重构**：引入 `react-resizable-panels` 彻底重构左右分屏组件，剥离了自研的 `style.width` 修改逻辑，依托底层的高效管理解决重排掉帧。
3. **超长文本虚拟化 (Windowing)**：针对纯文本提取区域，引入 `react-virtuoso` 实现了段落级别的虚拟列表渲染。现在无论解析提取了多少内容，DOM 树中只存在屏幕可视范围内的少数节点，滑动如丝般顺滑。
4. **智能混合文档按需渲染 (SmartDocViewer)**：
   - 针对 **PDF**：接入 `react-pdf` 配合 `Virtuoso`，实现真实的“滑动到哪一页渲染哪一页”懒加载机制，彻底解决多页大文件卡死问题。
   - 针对 **Word**：利用 CSS 新特性 `content-visibility: auto`，让浏览器自动跳过屏幕外排版节点的重绘与计算。

### 改动文件
- `frontend/package.json`: 引入 `framer-motion`, `react-resizable-panels`, `react-virtuoso`, `react-pdf` 核心性能依赖。
- `frontend/src/pages/Home.tsx` & `frontend/src/components/UploadBox.tsx`: 使用 motion API 重写大量出入场动画，并替换为 Group 面板拖拽系统。
- `frontend/src/components/UploadBox.tsx`: 使用 `Virtuoso` 重构 `HighlightText`。
- `frontend/src/components/SmartDocViewer.tsx`: 新建该组件，智能调度 PDF 和 Word 的性能优化渲染逻辑。
